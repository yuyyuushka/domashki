1. Семантическая верстка использует теги по смыслу содержимог, а не только для оформления. Важна для доступности, SEO и поддержки кода.
2. <article> для самостоятельного, независимого контента. <section> для тематической группировки внутри документа.
3. <header> и <footer> можно использовать для всего документа и для каждого структурного элемента (<article>, <section>).
4. Для основного контента используется <main>. <div> не несет смысловой нагрузки и ухудшает доступность и SEO.
5. <nav> семантически обозначает навигационные блоки. Обязателен для основных меню сайта. Обычный список без <nav> не имеет семантического значения.
6. datetime в <time> обеспечивает машинно-читаемый формат даты, что важно для поисковых систем.
7. <figure> и <figcaption> семантически связывают изображение с подписью, что улучшает доступность и структуру контента.
8. <aside> для дополнительного, косвенно связанного контента.
9. Семантическая разметка улучшает SEO (помогает поисковикам понять структуру контента) и доступность.
10. Используемые теги: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>. Они создают четкую структуру документа для пользователей и поисковых систем.
